## Project

WellD is an ESP32-C6 well monitor built with ESP-IDF. The ESP32-C6 is a RISC-V chip with Wi-Fi 6, BLE 5, and IEEE 802.15.4 (Zigbee/Thread).

## Commands

```bash
# First-time target setup
idf.py set-target esp32c6

# Build
idf.py build

# Flash and monitor (adjust port as needed)
idf.py -p /dev/ttyUSB0 flash monitor

# Monitor only
idf.py -p /dev/ttyUSB0 monitor

# Project configuration (menuconfig TUI)
idf.py menuconfig

# Clean build artifacts
idf.py fullclean
```

To run a single Unity-based component test:

```bash
idf.py -C test/<component> build flash monitor
```

## Architecture

ESP-IDF projects are composed of **components**. Each component lives in `components/<name>/` with its own `CMakeLists.txt` and `idf_component.yml` (for managed dependencies). The entry point is `main/` (itself a component).

Key files:
- `sdkconfig` — generated by `menuconfig`; commit this to pin config across machines
- `partitions.csv` — custom partition table if OTA or NVS sizing requires it
- `main/CMakeLists.txt` — lists source files for the main component
- `CMakeLists.txt` (root) — sets minimum IDF version and project name

Component dependencies are declared in `idf_component.yml` (fetched from the [IDF Component Registry](https://components.espressif.com/)) or in `CMakeLists.txt` via `REQUIRES` / `PRIV_REQUIRES`.

ESP32-C6 specific: use `CONFIG_IDF_TARGET_ESP32C6` guards for any chip-specific code. The chip has a single-core RISC-V at up to 160 MHz and a low-power RISC-V core for sleep tasks.
